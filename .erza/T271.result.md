## What was done (8 fixes)

1. **Accessibility statement page** — new `accessibility.html` (Hebrew, RTL),
   referencing IS 5568 / WCAG 2.1 AA and the Equal Rights for Persons with
   Disabilities Law.
2. **Accessibility coordinator** — named contact (Erez Cohen, E-REZULTS,
   support@erezults.com) inside `accessibility.html` §4.
3. **Accessibility widget** — new shared `accessibility-widget.js`, loaded on
   all 8 HTML pages (index, privacy, terms, support, accessibility,
   auth-callback, auth/confirm, join). Floating button + panel: text
   resize (4 steps via CSS `zoom`), high-contrast toggle (`filter: invert`),
   stop-animations toggle. Fully keyboard-operable (`<button>` elements,
   `Escape` to close, `aria-expanded`/`aria-pressed`/`role="dialog"`),
   prefs persisted in `localStorage`.
4. **Alt text audit** — verified: every `<img>` in the repo already has a
   meaningful `alt` (index, auth-callback, join, auth/confirm). No changes
   needed here.
5. **Color contrast** — `--text-muted` alpha bumped 0.3→0.5 (index.html,
   auth-callback.html, join/index.html) and `.header p` opacity bumped
   0.5→0.7 (privacy/terms/support.html). Verified by hand: old value was
   ~2.5:1 against the dark background (fails AA); new value is ~4.9:1
   (passes the 4.5:1 normal-text threshold).
6. **Keyboard navigation** — added global `a:focus-visible, button:focus-visible`
   outline rules on every page; removed `outline: none` on the
   password-reset inputs in `auth/confirm/index.html` and replaced it with
   a visible `:focus-visible` ring; added a skip-to-content link on
   `index.html`. All interactive elements site-wide were already native
   `<button>`/`<a>` (no custom click-only divs), so no tabindex/role fixes
   were needed beyond this.
7. **Cookie/local-storage notice** — new §6 in `privacy.html` disclosing
   that the site uses `localStorage` (not cookies) for language and
   accessibility-widget preferences only, stays on-device, and that Google
   Fonts may set its own cookies. Sections renumbered 6→11 accordingly;
   "Last updated" bumped to July 14, 2026.
8. **Footer discoverability** — added an "Accessibility Statement" link to
   the footer/back-link of index.html, privacy.html, terms.html, and
   support.html.
